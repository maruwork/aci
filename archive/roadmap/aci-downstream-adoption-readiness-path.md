# ACI Downstream Adoption Readiness Path

Status: Complete

## Path

1. inventory current downstream-adoption language
2. define the bounded downstream adoption model
3. implement the common downstream adoption packet surface
4. document downstream ownership boundaries and reading route
5. verify the reviewed adoption surface

## Stop When

- adoption metadata starts requiring one specific downstream repository layout
- the common shelf starts owning project-local runtime or approval logic
- the adoption packet cannot stay generic without project-local assumptions
