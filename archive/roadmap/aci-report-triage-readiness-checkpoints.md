# ACI Report Triage Readiness Checkpoints

Status: Complete

## RTR-CP1 Inventory

Pass when:
- current report contracts and sample reports are inventoried
- missing triage surfaces are explicit

## RTR-CP2 Triage Field Set

Pass when:
- the minimum generic triage fields are fixed
- field ownership is split between finding row, summary, blocker, and residual sections

## RTR-CP3 Finding Helper Update

Pass when:
- normalized finding helpers can carry the triage fields required by the contracts

## RTR-CP4 Contract Update

Pass when:
- human-readable and machine-readable contracts define the same triage model

## RTR-CP5 Sample Update

Pass when:
- sample reports use the revised triage fields consistently
- examples stay generic and neutral

## RTR-CP6 Reader Guidance

Pass when:
- report-facing reading docs explain how to use the new triage fields

## RTR-CP7 Final Review

Pass when:
- reviewed report surfaces help an operator distinguish fix-now, review-first, and accepted-residual work
