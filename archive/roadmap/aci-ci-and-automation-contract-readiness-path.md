# ACI CI And Automation Contract Readiness Path

Status: Complete

## Path

1. inventory current automation-capable surfaces
2. define the minimum automation contract
3. implement bounded automation commands or flags
4. document CI-facing output and exit behavior
5. verify repeatable automation use

## Stop Conditions

- automation behavior starts requiring downstream workflow ownership
- contract becomes provider-specific
- a required feature belongs to a later fixture or state-lifecycle wave
