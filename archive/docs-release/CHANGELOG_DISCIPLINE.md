# ACI Changelog Discipline

Status: Active

## Purpose

Make release summary writing repeatable and product-oriented.

## Required Sections

- `Summary`
  - what changed in one short paragraph
- `Why`
  - why the change was needed at the product level
- `User-Facing Impact`
  - CLI, config, report, contract, or adoption effect
- `Verification`
  - what was run
- `Compatibility Notes`
  - whether existing usage stays valid

## Change Classification

Every release candidate should say whether the change is primarily:

- feature
- fix
- contract clarification
- compatibility improvement
- release-prep improvement

## Not This Discipline

This file does not own hosted release-note formatting or provider-specific markdown conventions.
